### E. 問題陳述（Problem Statement）

```
對於需要進行課程小組報告的大學生與小組組長而言，目前在傳統的自由組隊與人工分工情境中，常發生邊緣學生媒合困難、成員修課目標不一導致擺爛雷友出現，以及組長難以客觀催繳進度與留存分工證據的痛點與影響。

本專案希望設計一個課程分組媒合與任務進度追蹤系統，協助學生依據技能與修課期待自主媒合隊友、提供組長數位化的任務指派與催繳進度看板，並在本期聚焦於個人檔案與技能建立、隊友尋找媒合看板、小組報告任務指派與進度狀態變更，不處理即時聊天室實作與校務選課系統的正式 API 串接。
```

### F. 專案目標（Project Goal）

```
本專案旨在建立一個兼具隊友媒合與任務追蹤之數位化協作平台，透過公開的技能期待看板與可追溯的分工進度軌跡，降低大學小組報告中資訊不對稱與組員擺爛之風險，進而提升團隊組隊效率與分工催繳之透明度。
```

### G. 範圍內項目（In-Scope）

至少 5 項。

1. 使用者角色與基本權限管理
2. 個人技能與期待檔案維護
3. 尋找隊友公開媒合看板小組
4. 小組報告專案建立與成員管理
5. 小組任務指派功能
6. 任務進度回報與狀態變更流程
7. 組長審核與催繳日誌

### H. 範圍外項目（Out-of-Scope）

至少 5 項。

1. 校務教務系統正式選課 API 串接
2. 即時線上聊天室與語音通訊功能
3. 雲端檔案儲存與即時共同編輯功能
4. 自動化智慧配對與推薦演算法
5. 同儕期末互評分數之正式結算與匯出
6. 行動裝置原生應用程式（Native App）開發
7. 正式環境部署與伺服器上線營運
