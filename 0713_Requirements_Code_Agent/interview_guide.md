# 訪談大綱（Interview Guide）

## 小組基本資料

- 小組名稱：Group2
- 專案題目：智慧課程分組與跨系組隊媒合系統
- 文件版本：v1
- 日期：115/07/13

## 角色 1：一般學生 / 邊緣人

| 欄位 | 內容 |
| --- | --- |
| 受訪角色 | 一般學生 / 邊緣人 |
| 角色與系統關係 | 主要操作者，透過系統尋找隊友與填寫技能標籤 |
| 訪談目的 | 了解學生在課堂分組時的痛點、希望的媒合方式與期望的資訊 |

| 題號 | 訪談問題 | 想確認的需求或風險 |
| --- | --- | --- |
| 1 | 你平常在課堂或專題中，怎麼找到隊友？ | 了解目前分組方式與痛點 |
| 2 | 你最常遇到哪一種組隊困難？ | 確認落單、跨系、技能不互補等情境 |
| 3 | 你會希望系統在分組前先知道哪些個人資訊？ | 確認技能標籤與偏好欄位 |
| 4 | 如果有系統可幫你篩選隊伍，你最在意哪些條件？ | 確認媒合規則與優先條件 |
| 5 | 若你加入某隊伍但後來不適合，現在或未來會如何處理？ | 確認退出、轉隊或重新申請流程 |
| 6 | 你認為一個好用的組隊系統，最重要的成功標準是什麼？ | 確認成功標準與期望價值 |

## 角色 2：專案組長 / 隊伍管理者

| 欄位 | 內容 |
| --- | --- |
| 受訪角色 | 專案組長 / 隊伍管理者 |
| 角色與系統關係 | 需要建立隊伍、編排任務與追蹤組員進度 |
| 訪談目的 | 了解組長對任務看板與分工透明化的需求 |

| 題號 | 訪談問題 | 想確認的需求或風險 |
| --- | --- | --- |
| 1 | 你目前如何管理組內任務與分工？ | 了解現況流程與痛點 |
| 2 | 你最常遇到哪一種組內協作問題？ | 確認任務遺漏、進度不透明與搭便車問題 |
| 3 | 你希望任務看板至少包含哪些狀態？ | 確認 Todo / Doing / Done 的必要性 |
| 4 | 如果某位成員長期沒完成任務，現在要如何處理？ | 確認預警與追蹤需求 |
| 5 | 你會希望系統在組長建立任務時自動提醒哪些資訊？ | 確認資料需求與提醒邏輯 |
| 6 | 你認為一個可接受的任務追蹤流程，應該讓誰看到哪些內容？ | 確認權限與可見性需求 |
