# 程式碼代理實作任務書（Code Agent Implementation Brief）

## 0. 工具選擇

| 項目 | 內容 |
| --- | --- |
| 本組使用的程式碼代理（code agent） | GitHub Copilot / VS Code 內建代理 |
| 使用介面 | Visual Studio Code |
| 選擇原因 | 方便直接修改專案檔案、檢查 HTML/CSS/JavaScript 並保留版本記錄 |
| 是否需要登入或授權 | 是，需使用 VS Code 與 GitHub 帳號 |
| 是否會上傳專案內容到外部服務 | 會依賴工具內建服務，但本次不會上傳敏感資料 |
| 小組如何避免外送不該外送的資料 | 只提供課堂需求與假資料，不提供真實學生資料 |

## 1. 專案背景

本專案為「智慧課程分組與跨系組隊媒合系統」的第一個可操作切片，目標是在不連接正式資料庫與正式登入的前提下，讓使用者先體驗「學生填寫技能、查看隊伍、建立任務」的核心流程。

## 2. 目標角色

| 角色 | 主要任務 | 主要痛點 |
| --- | --- | --- |
| 一般學生 | 填寫技能標籤、查看隊伍資訊、提交加入申請 | 不知道哪個隊伍適合自己，流程容易尷尬 |
| 專案組長 | 建立任務、查看進度、更新狀態 | 任務分工不透明，成員進度難追蹤 |

## 3. 第一個可操作切片（First Working Slice）

本次只做「學生查看隊伍並提交加入申請；組長建立任務並更新狀態」的最小流程。

## 4. 本期範圍內項目（In-Scope）

- 學生填寫技能標籤與偏好
- 學生查看隊伍資訊與缺口
- 學生提交加入申請
- 組長建立任務
- 任務狀態切換（Todo / Doing / Done）
- 使用假資料與靜態畫面

## 5. 本期範圍外項目（Out-of-Scope）

- 正式登入
- 正式資料庫
- 即時聊天
- 金流
- 真實智慧自動配對演算法

## 6. 本次要實作的需求

| 需求編號 | 來源編號 | 需求描述 | 驗收條件 | 對應畫面或操作 |
| --- | --- | --- | --- | --- |
| FR-01 | INT-STU-01 | 系統應讓學生輸入技能標籤與偏好。 | 學生可在表單中填寫技能與偏好，並在畫面中顯示。 | 學生 Profile 畫面 |
| FR-02 | INT-STU-02 | 系統應顯示隊伍目前人數與缺口。 | 隊伍卡片顯示人數、缺口與需求角色。 | 隊伍列表畫面 |
| FR-04 | INT-GL-01 | 系統應提供任務清單與狀態。 | 組長可新增任務並看到任務狀態。 | 任務看板畫面 |
| NFR-01 | NFR-01 | 系統介面應清楚且易理解。 | 主要操作按鈕與狀態標籤清楚可辨。 | 全站介面 |

## 7. 技術限制（Technical Constraints）

| 限制項目 | 本組設定 |
| --- | --- |
| 前端技術 | HTML + CSS + JavaScript |
| 後端技術 | 不使用 |
| 資料儲存 | 使用假資料（mock data） |
| 是否需要正式部署 | 否 |
| 是否允許外部套件 | 僅可使用原生或最小必要套件 |
| 是否允許正式登入 | 否 |

## 8. 程式碼代理不可自行決定的事項

- 不可自行新增正式登入或資料庫功能。
- 不可新增即時聊天、金流或校務系統整合。
- 不可將本次切片擴大為完整系統。

## 9. 若代理想加入，必須拒絕的功能

| 功能 | 拒絕理由 | 對應範圍外項目 |
| --- | --- | --- |
| 正式登入流程 | 會超出本期範圍且不符需求限制 | 正式登入 |
| 即時聊天功能 | 本次切片聚焦於媒合與任務看板，不需要聊天 | 即時通訊 |
| 真實智慧自動配對演算法 | 技術與時程風險過高，且已移出範圍 | 智慧演算法 |

## 10. 交給程式碼代理的正式任務文字

```text
請依照以下系統分析與設計資料，建立第一個可操作切片。

專案背景：智慧課程分組與跨系組隊媒合系統的第一個切片，重點是學生查看隊伍與提交加入申請，組長建立任務並切換狀態。

目標角色：一般學生、專案組長。

第一個可操作切片：學生查看隊伍並提交加入申請；組長建立任務並更新狀態。

本次要支援的需求編號：FR-01、FR-02、FR-04、NFR-01。

本期範圍內：技能填寫、隊伍資訊、加入申請、任務建立、任務狀態切換。

本期範圍外：正式登入、正式資料庫、即時聊天、金流、真實智慧演算法。

技術限制：使用 HTML、CSS、JavaScript，使用假資料，不使用正式登入與資料庫。

驗收條件：學生可填寫技能與偏好、查看隊伍缺口並提交申請；組長可新增任務並切換狀態。

不可捏造事項：不要新增校務系統整合、正式登入或即時通訊。

請先產生實作計畫，確認後再開始建立或修改檔案。
```
