# 更新後的程式碼代理實作任務書

## 一、專案與目前狀態
- 專案名稱：智慧課程分組與跨系組隊媒合系統
- 目前狀態：已完成第二個可操作切片，並延伸為第三個可操作切片，支援學生偏好、申請加入與任務狀態切換。

## 二、本次使用案例與目標
- 使用案例：UC-01 儲存學生偏好、UC-03 提交加入申請、UC-04 建立任務、UC-05 更新任務狀態
- 目標：讓學生與組長透過介面完成核心流程，並在操作後看到結果與狀態。

## 三、本次要完成的流程
- 主要流程：學生輸入偏好、隊伍申請、任務建立與狀態更新
- 替代流程：隊伍已額滿時停用申請，僅顯示已額滿
- 例外流程：空白輸入與重複申請時顯示錯誤或現況狀態

## 四、前置與後置條件
- 前置：學生進入學生區塊、組長進入任務看板
- 後置：畫面顯示偏好、申請狀態與任務狀態

## 五、輸入、輸出與狀態變化
- 輸入：技能標籤、興趣與偏好、隊伍編號、任務名稱、負責人
- 輸出：偏好清單、申請紀錄、任務卡片、提示訊息
- 狀態變化：空白→已儲存、未申請→申請中、Todo→Doing/Done

## 六、保留項目
- 保留學生、組長、老師角色的基本介面
- 保留假資料與繁體中文介面

## 七、修正項目
- 修正偏好空白驗證與提示
- 修正申請加入重複防護
- 修正任務新增與狀態切換邏輯

## 八、新增項目
- 新增申請狀態顯示與重複提交防護
- 新增任務狀態變更與提示

## 九、刪除項目
- 不新增正式登入、金流、外部通知與正式資料庫

## 十、本次不做
- 正式登入

## 十一、技術與資料限制
