# 0720 使用案例與流程模型實作

## 今日分析範圍
- 基準需求版本：0714 使用者故事、功能性需求、驗收條件與第二個可操作切片
- 專案名稱：智慧課程分組與跨系組隊媒合系統
- 主要分析範圍：學生偏好輸入、隊伍申請、任務看板與簡易摘要

## 主要參與者與使用案例
- 參與者：一般學生、專案組長、授課老師
- 核心使用案例：儲存學生偏好、查看隊伍與缺口、提交加入申請、建立任務、更新任務狀態、查看簡易摘要

## 現況問題與目標改善摘要
- 現況流程中，學生申請與隊伍狀態資訊分散，容易造成重複詢問與狀態不透明。
- 目標流程將申請、狀態與任務更新整合到系統中，保留必要人工審核與角色責任。

## 本次實作選擇
