# 使用案例描述

## UC-01 儲存學生偏好
- 目標：讓學生建立個人技能與偏好，供後續隊伍參考。
- 主要參與者：一般學生
- 支援參與者：無
- 觸發事件：學生於學生區塊輸入技能或偏好。
- 前置條件：學生進入學生區塊。
- 成功後置條件：偏好內容顯示在畫面中，且後續可在摘要中被重複使用。
- 最低保證：空白輸入不建立空白偏好，系統顯示錯誤訊息。
- 主要流程：1. 學生輸入技能與偏好；2. 系統檢查是否至少有一項內容；3. 系統顯示已儲存訊息；4. 畫面更新為最新偏好。
- 替代流程 A1：學生僅輸入一項內容；系統仍可儲存，並顯示該項內容。
- 例外流程 E1：學生未輸入任何內容；系統顯示錯誤訊息，不建立偏好。
- 輸入資料：技能標籤、興趣與偏好
- 輸出資料：偏好清單
- 狀態變化：空白→已儲存
- 對應需求：FR-01
- 對應驗收條件：AC-01-01、AC-01-02

## UC-03 提交加入申請
- 目標：讓學生向隊伍提交加入申請，並取得明確狀態。
- 主要參與者：一般學生
- 支援參與者：專案組長
- 觸發事件：學生於隊伍卡片點擊「申請加入」。
- 前置條件：隊伍存在且尚未額滿，學生尚未送出同隊伍申請。
- 成功後置條件：建立一筆申請紀錄，並顯示「已送出申請」。
- 最低保證：重複申請不建立第二筆紀錄，系統顯示現況狀態。
- 主要流程：1. 學生查看隊伍資訊；2. 系統顯示隊伍缺口與狀態；3. 學生點擊申請；4. 系統檢查是否已額滿與是否已有申請；5. 系統建立申請紀錄；6. 系統顯示申請狀態。
- 替代流程 A1：隊伍已額滿；系統停用申請按鈕並顯示已額滿。
- 例外流程 E1：學生已提交相同隊伍申請；系統顯示目前狀態，不建立新申請。
- 輸入資料：隊伍編號、學生識別
- 輸出資料：申請紀錄、申請狀態
- 狀態變化：未申請→申請中
- 對應需求：FR-03
- 對應驗收條件：AC-03-01、AC-03-02

## UC-04 建立任務
- 目標：讓組長建立任務並顯示於任務看板。
- 主要參與者：專案組長
- 支援參與者：無
- 觸發事件：組長輸入任務名稱與負責人並點擊新增。
- 前置條件：組長進入任務看板。
- 成功後置條件：任務出現在清單中，狀態為 Todo。
- 最低保證：任務名稱或負責人空白時不建立任務，且顯示錯誤訊息。
- 主要流程：1. 組長輸入任務名稱與負責人；2. 系統檢查欄位是否為空白；3. 系統建立任務；4. 任務顯示於看板。
- 替代流程 A1：組長可選擇初始狀態為 Doing；系統仍將任務建立於看板。
- 例外流程 E1：任務名稱或負責人缺漏；系統不建立任務並顯示提示。
- 輸入資料：任務名稱、負責人、初始狀態
- 輸出資料：任務卡片
- 狀態變化：無任務→已建立 Todo
- 對應需求：FR-04
- 對應驗收條件：AC-04-01、AC-04-02
