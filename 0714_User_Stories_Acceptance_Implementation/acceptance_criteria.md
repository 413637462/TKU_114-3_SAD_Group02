# 驗收條件（Acceptance Criteria）

| 驗收編號 | 對應需求 | 情境類型 | 前提 | 當 | 那麼 | 可重現 |
| --- | --- | --- | --- | --- | --- | --- |
| AC-01-01 | FR-01 | 正常 | 學生進入學生區塊 | 學生輸入技能標籤與偏好並點擊「儲存偏好」 | 系統在畫面中顯示已輸入內容 | 是 |
| AC-01-02 | FR-01 | 例外 | 學生未輸入任何內容 | 學生點擊「儲存偏好」 | 系統顯示提示訊息，且不建立空白偏好 | 是 |
| AC-02-01 | FR-02 | 正常 | 隊伍資料已存在 | 學生查看隊伍卡片 | 系統顯示隊伍人數、缺口與需求角色 | 是 |
| AC-02-02 | FR-02 | 邊界 | 隊伍已滿員 | 學生查看隊伍卡片 | 系統顯示「已額滿」狀態並停用申請按鈕 | 是 |
| AC-03-01 | FR-03 | 正常 | 學生已選擇隊伍 | 學生點擊「申請加入」 | 系統顯示「申請已送出」並保留申請狀態 | 是 |
| AC-03-02 | FR-03 | 例外 | 學生已提交過相同隊伍申請 | 學生再次點擊「申請加入」 | 系統顯示目前申請狀態，不重複建立新申請 | 是 |
| AC-04-01 | FR-04 | 正常 | 組長進入任務看板 | 組長輸入任務名稱與負責人並點擊新增 | 任務出現在任務清單中 | 是 |
| AC-04-02 | FR-04 | 邊界 | 任務名稱為空 | 組長點擊新增 | 系統顯示錯誤訊息且不建立任務 | 是 |
| AC-05-01 | FR-06 | 正常 | 任務已建立 | 組長點擊「切換為 Doing」 | 任務狀態由 Todo 更新為 Doing | 是 |
| AC-05-02 | FR-06 | 邊界 | 任務目前為 Done | 組長嘗試再次切換 | 系統顯示目前狀態，且不產生錯誤的重複切換 | 是 |
| AC-NFR-01-01 | NFR-01 | 品質 | 使用者第一次操作原型 | 使用者進行主要操作 | 主要按鈕與狀態標籤清楚可辨，並能在一個畫面內完成主要流程 | 是 |
