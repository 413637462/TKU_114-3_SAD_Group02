# 更新後的程式碼代理實作任務書

## 一、專案名稱
智慧課程分組與跨系組隊媒合系統

## 二、目前系統狀態
- 已有第一個可操作切片，包含學生區塊、隊伍資訊與組長任務看板。
- 目前為靜態 HTML/JavaScript 原型，使用假資料。

## 三、本次目標
- 完成第二個可操作切片：讓學生可輸入偏好、提交加入申請，並讓組長可建立任務與更新狀態。

## 四、保留項目
- 保留學生與組長兩個角色的基本介面。
- 保留原有之假資料與繁體中文介面文字。

## 五、修正項目
- 修正學生偏好輸入與顯示邏輯。
- 修正隊伍卡片顯示與額滿狀態。
- 修正申請加入流程與重複送出防護。
- 修正任務新增與狀態切換邏輯。

## 六、新增項目
- 新增偏好儲存與提示訊息。
- 新增申請狀態顯示與動態更新。
- 新增任務建立與狀態切換互動。

## 七、刪除項目
- 移除不必要的範圍外功能，例如正式登入與即時聊天。

## 八、本次不做
- 正式登入
- 正式資料庫
- 即時聊天
- 金流
- 真實自動配對演算法

## 九、技術與資料限制
- 使用 HTML/CSS/JavaScript
- 使用假資料與瀏覽器暫存資料
- 不串接正式外部服務

## 十、對應需求與驗收條件
- FR-01：AC-01-01、AC-01-02
- FR-02：AC-02-01、AC-02-02
- FR-03：AC-03-01、AC-03-02
- FR-04、FR-06：AC-04-01、AC-05-01
- NFR-01：AC-NFR-01-01

## 十一、實作順序
1. 建立學生偏好輸入與訊息提示
2. 建立隊伍資訊與額滿狀態顯示
3. 建立申請加入流程
4. 建立任務新增與狀態切換
5. 進行手動驗收與修正

## 十二、不可自行決定的事項
- 不可新增正式認證或資料庫整合
- 不可擴充為完整分組管理系統
- 不可替換需求來源與驗收條件

## 十三、完成後回報格式
1. 修改檔案與用途
2. 操作方式
3. 需求與驗收條件對應
4. 已通過與未通過項目
5. 已知限制
6. 未完成項目
